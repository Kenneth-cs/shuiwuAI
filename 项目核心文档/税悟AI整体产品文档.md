## 一、 产品需求文档 (PRD)

### 1. 目标与范围

* **核心目标**：验证 AI 对财务原始数据的理解能力，并准确填充至广东省电子税务局要求的报表格式中。
* **首批覆盖报表**：
1. 《增值税及附加税费申报表（小规模纳税人适用）》
2. 《居民企业（查账征收）企业所得税月（季）度预缴纳税申报表（A类）》
3. 《文化事业建设费申报表》



### 2. 页面功能逻辑

#### 模块一：智能数据采集页 (Input)

* **多模态输入**：
* **对话式输入**：老板输入“本月开票收入 5 万，其中 3 万是 1% 税率，2 万免税；支出办公费 2000 元”。
* **文件上传**：支持上传你提供的《模版-财务报表.xls》（利润表、资产负债表、现金流量表）。


* **AI 预处理**：
* 调用火山引擎 OCR 识别财务报表数值。
* 调用 LLM 将自然语言/非标 Excel 数据结构化，映射为标准财税科目。


* **数据确认**：在 AI 生成结果后，提供一个简易列表让用户确认：收入、成本、利润、从业人数等核心数值。

#### 模块二：辅助报税导航页 (Output)

* **双栏对比视图**：
* **左侧（系统指引）**：显示生成的报表缩略图，并用高亮/气泡标注：*“请在税务局第 1 行填入 50000.00”*。
* **右侧（内嵌/跳转说明）**：提供对应广东税务局操作界面的截图指引或操作链接。


* **报表导出**：支持生成与 Excel 模板一致的 PDF 或图片，方便用户核对。

### 3. 业务规则（逻辑要点）

* **增值税逻辑**：需识别小规模纳税人月销售额是否超过 10 万（或季 30 万）的免税政策（根据 2026 年实时政策调整）。
* **所得税逻辑**：基于《利润表》的“利润总额”计算，并自动判定“小型微利企业”优惠（资产总额、从业人数参考你提供的 A200000 表格）。
* **文化事业建设费**：识别“广告业”或“娱乐业”收入，按 3%（或当前费率）计算应缴费额。

## 二、 技术开发文档

### 1. 技术栈

* **前端**：React / Next.js (适配 Web 端)
* **后端**：Python (FastAPI) —— 易于集成 AI 模块
* **数据库**：MySQL 8.0
* **AI 能力**：**火山引擎 (Volcano Engine)**
* **LLM**: 豆包 (Doubao-pro) 用于数据推理与报表映射。
* **OCR**: 用于提取用户上传的财务报表截图或 Excel 截图。



### 2. 数据库设计 (MySQL)

设计核心表以存储原始数据与 AI 处理后的映射结果。

```sql
-- 企业基本信息表
CREATE TABLE companies (
    id INT PRIMARY KEY AUTO_INCREMENT,
    company_name VARCHAR(255),
    tax_id VARCHAR(50), -- 纳税人识别号
    tax_type ENUM('小规模纳税人', '一般纳税人'),
    industry VARCHAR(100) -- 用于判断文化事业建设费
);

-- 财务原始数据表 (Demo期核心)
CREATE TABLE financial_raw_data (
    id INT PRIMARY KEY AUTO_INCREMENT,
    company_id INT,
    period VARCHAR(7), -- 2025-12
    raw_content TEXT, -- 存储AI解析前的JSON字符串
    source_type EN_TYPE('chat', 'excel', 'ocr'),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 报表映射结果表
CREATE TABLE tax_report_snapshots (
    id INT PRIMARY KEY AUTO_INCREMENT,
    raw_data_id INT,
    report_type VARCHAR(50), -- VAT, CIT, CULTURE_FEE
    json_data JSON, -- 存储各表单字段的具体数值
    status ENUM('draft', 'confirmed')
);

```

### 3. AI 逻辑流程 (火山引擎集成)

#### Step 1: 提示词工程 (Prompt Engineering)

针对你提供的 Excel 模板，构建系统级 Prompt。

> **System Prompt**: 你是一个精通广东省税务政策的专家。我会给你一份结构化的财务数据（JSON格式）。请根据《小企业会计准则》和广东税务局申报要求，计算出《增值税及附加税费申报表》和《企业所得税预缴纳税申报表》的每一个行次对应数值。输出格式必须为 JSON，key 必须对应报表的行次编号（如：line_1, line_2）。

#### Step 2: 接口调用逻辑

1. **数据清洗**：后端接收输入后，将 Excel 或对话文字汇总，构造一个统一的 `Financial_Context`。
2. **火山引擎请求**：
* 模型选择：`Doubao-pro-32k` (适合处理复杂表格关系)。
* 流式返回结果，前端实时显示 AI 正在计算各个税种。



### 4. 辅助报税映射逻辑

为了实现 UI 设计稿中的“导航”功能，需要建立一个 **“报表字段 -> 税务局网页元素”** 的映射库：

* **Mapping 示例**：
* `VAT_Report.line_1` (本期销售额) -> 对应广东电子税务局申报页面的 ID `input_001`。
* `CIT_Report.line_28.1` (小微企业减免) -> 对应 ID `input_028_1`。
